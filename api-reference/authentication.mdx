## Overview

The Evento Public API uses **Bearer token authentication** powered by [unkey.dev](https://unkey.dev). Every request must include your API key in the `Authorization` header.

<Warning>
  Never expose your API key in client-side code or public repositories. Always use environment variables and server-side requests.
</Warning>

## Authentication Header

Include your API key in the `Authorization` header of every request:

```
Authorization: Bearer evento_xxxxxxxxxxxxxxxxxxxxxxxx
```

## Example Requests

<CodeGroup>

```bash cURL
curl https://api.evento.so/public/v1/events/evt_abc123 \
  -H "Authorization: Bearer evento_xxxxxxxxxxxxxxxxxxxxxxxx"
```

```javascript JavaScript
const response = await fetch('https://api.evento.so/public/v1/events/evt_abc123', {
  headers: {
    'Authorization': `Bearer ${process.env.EVENTO_API_KEY}`
  }
});

const data = await response.json();
```

```python Python
import os
import requests

headers = {
    'Authorization': f'Bearer {os.environ["EVENTO_API_KEY"]}'
}

response = requests.get(
    'https://api.evento.so/public/v1/events/evt_abc123',
    headers=headers
)

data = response.json()
```

```go Go
package main

import (
    "fmt"
    "net/http"
    "os"
)

func main() {
    apiKey := os.Getenv("EVENTO_API_KEY")
    
    req, _ := http.NewRequest("GET", "https://api.evento.so/public/v1/events/evt_abc123", nil)
    req.Header.Set("Authorization", fmt.Sprintf("Bearer %s", apiKey))
    
    client := &http.Client{}
    resp, _ := client.Do(req)
    defer resp.Body.Close()
}
```

</CodeGroup>

## Error Responses

### 401 Unauthorized

Returned when the API key is missing, invalid, or revoked:

<ResponseExample>
```json
{
  "success": false,
  "message": "Invalid or missing API key"
}
```
</ResponseExample>

**Common causes:**
- Missing `Authorization` header
- Invalid API key format
- Revoked or expired API key
- API key from a different environment

### 429 Too Many Requests

Returned when you exceed the rate limit (1,000 requests/day):

<ResponseExample>
```json
{
  "success": false,
  "message": "Rate limit exceeded. Try again after reset time."
}
```
</ResponseExample>

**Response headers include:**
```
X-RateLimit-Limit: 1000
X-RateLimit-Remaining: 0
X-RateLimit-Reset: 1640995200
```

## Best Practices

<CardGroup cols={2}>
  <Card title="Use Environment Variables" icon="shield">
    Store API keys in environment variables, never hardcode them
    
    ```bash
    EVENTO_API_KEY=evento_xxx
    ```
  </Card>
  
  <Card title="Server-Side Only" icon="server">
    Make API requests from your backend, not from client-side JavaScript
  </Card>
  
  <Card title="Handle Errors Gracefully" icon="triangle-exclamation">
    Always check for 401 and 429 responses and handle them appropriately
  </Card>
  
  <Card title="Implement Retry Logic" icon="rotate">
    Use exponential backoff when retrying failed requests
  </Card>
</CardGroup>

## Testing Your Authentication

Use this simple request to verify your API key is working:

<RequestExample>
```bash cURL
curl https://api.evento.so/public/v1/events/evt_test \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -v
```
</RequestExample>

<Tip>
  A successful authentication will return a `200` status code (even if the event doesn't exist, you'll get a `404` instead of `401`).
</Tip>
