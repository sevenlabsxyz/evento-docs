Retrieve all upcoming public events hosted by a specific user, sorted chronologically.

<Info>
This endpoint returns only **future events** created by the user, sorted by start date (earliest first). Perfect for displaying a user's event schedule.
</Info>

## Endpoint

```
GET https://api.evento.so/api/public/v1/users/{username}/upcoming-events
```

## Path Parameters

<ParamField path="username" type="string" required>
  The username of the Evento user
  
  Example: `johndoe`
</ParamField>

## Query Parameters

<ParamField query="limit" type="integer" min="1" max="100" default="10">
  Number of results to return per page
  
  - Minimum: 1
  - Maximum: 100
  - Default: All upcoming events (no limit)
  
  Example: `10`
</ParamField>

<ParamField query="offset" type="integer" min="0" default="0">
  Number of results to skip for pagination
  
  - Minimum: 0
  - Default: 0
  
  Example: `20` (skip first 20 results)
</ParamField>

## Headers

<ParamField header="x-evento-api-key" type="string" required>
  Your API key for authentication
</ParamField>

## Examples

<CodeGroup>

```bash cURL
curl -X GET "https://api.evento.so/api/public/v1/users/johndoe/upcoming-events?limit=5" \
  -H "x-evento-api-key: YOUR_API_KEY"
```

```javascript JavaScript
const response = await fetch(
  'https://api.evento.so/api/public/v1/users/johndoe/upcoming-events?limit=10&offset=0',
  {
    method: 'GET',
    headers: {
      'x-evento-api-key': 'YOUR_API_KEY'
    }
  }
);

const result = await response.json();

if (result.success) {
  console.log(`Found ${result.data.length} upcoming events`);
  
  result.data.forEach(event => {
    const date = new Date(event.start_date);
    console.log(`- ${event.title} on ${date.toLocaleDateString()}`);
  });
}
```

```python Python
import requests
from datetime import datetime

url = "https://api.evento.so/api/public/v1/users/johndoe/upcoming-events"
params = {
    "limit": 10,
    "offset": 0
}
headers = {
    "x-evento-api-key": "YOUR_API_KEY"
}

response = requests.get(url, params=params, headers=headers)
result = response.json()

if result["success"]:
    print(f"Found {len(result['data'])} upcoming events")
    
    for event in result["data"]:
        start = datetime.fromisoformat(event["start_date"].replace("Z", "+00:00"))
        print(f"- {event['title']} on {start.strftime('%B %d, %Y')}")
```

```php PHP
<?php
$curl = curl_init();

$params = http_build_query([
    'limit' => 10,
    'offset' => 0
]);

curl_setopt_array($curl, array(
  CURLOPT_URL => 'https://api.evento.so/api/public/v1/users/johndoe/upcoming-events?' . $params,
  CURLOPT_RETURNTRANSFER => true,
  CURLOPT_HTTPHEADER => array(
    'x-evento-api-key: YOUR_API_KEY'
  ),
));

$response = curl_exec($curl);
curl_close($curl);

$result = json_decode($response, true);

if ($result['success']) {
    echo "Found " . count($result['data']) . " upcoming events\n";
    
    foreach ($result['data'] as $event) {
        $date = new DateTime($event['start_date']);
        echo "- {$event['title']} on " . $date->format('F j, Y') . "\n";
    }
}
?>
```

## Notes

<Note>
- **Upcoming** means `start_date >= current_time`
- Events are sorted by start date in **ascending order** (earliest first)
- Only events **created by** the specified user are returned
- Empty array is returned if the user has no upcoming events
- Use pagination parameters for large result sets
</Note>

<Tip>
Combine `limit` and `offset` for efficient pagination. For example:
- Page 1: `limit=10&offset=0`
- Page 2: `limit=10&offset=10`
- Page 3: `limit=10&offset=20`
</Tip>
