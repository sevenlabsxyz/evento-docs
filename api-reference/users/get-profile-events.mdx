Retrieve all public events that a user has either created or RSVP'd to attend.

<Info>
This endpoint combines both hosted events and events the user is attending, making it perfect for displaying a user's complete event activity profile.
</Info>

## Endpoint

```
GET https://api.evento.so/api/public/v1/users/{username}/profile-events
```

## Path Parameters

<ParamField path="username" type="string" required>
  The username of the Evento user
  
  Example: `johndoe`
</ParamField>

## Query Parameters

<ParamField query="since" type="string" format="date-time">
  Filter events starting from this date (inclusive)
  
  Format: ISO 8601 (`YYYY-MM-DDTHH:mm:ssZ`)
  
  Example: `2024-01-01T00:00:00Z`
</ParamField>

<ParamField query="from" type="string" format="date-time">
  Alias for `since` parameter
  
  Format: ISO 8601 (`YYYY-MM-DDTHH:mm:ssZ`)
</ParamField>

<ParamField query="to" type="string" format="date-time">
  Filter events up to this date (inclusive)
  
  Format: ISO 8601 (`YYYY-MM-DDTHH:mm:ssZ`)
  
  Example: `2024-12-31T23:59:59Z`
</ParamField>

## Headers

<ParamField header="x-evento-api-key" type="string" required>
  Your API key for authentication
</ParamField>

## Examples

<CodeGroup>

```bash cURL
curl -X GET "https://api.evento.so/api/public/v1/users/johndoe/profile-events" \
  -H "x-evento-api-key: YOUR_API_KEY"
```

```javascript JavaScript
const response = await fetch(
  'https://api.evento.so/api/public/v1/users/johndoe/profile-events',
  {
    method: 'GET',
    headers: {
      'x-evento-api-key': 'YOUR_API_KEY'
    }
  }
);

const result = await response.json();

if (result.success) {
  const hostedEvents = result.data.filter(
    event => event.creator.username === 'johndoe'
  );
  const attendingEvents = result.data.filter(
    event => event.creator.username !== 'johndoe'
  );
  
  console.log(`Hosting: ${hostedEvents.length} events`);
  console.log(`Attending: ${attendingEvents.length} events`);
}
```

```python Python
import requests

url = "https://api.evento.so/api/public/v1/users/johndoe/profile-events"
params = {
    "since": "2024-01-01T00:00:00Z"
}
headers = {
    "x-evento-api-key": "YOUR_API_KEY"
}

response = requests.get(url, params=params, headers=headers)
result = response.json()

if result["success"]:
    events = result["data"]
    hosted = [e for e in events if e["creator"]["username"] == "johndoe"]
    attending = [e for e in events if e["creator"]["username"] != "johndoe"]
    
    print(f"Hosting: {len(hosted)} events")
    print(f"Attending: {len(attending)} events")
```

```php PHP
<?php
$curl = curl_init();

curl_setopt_array($curl, array(
  CURLOPT_URL => 'https://api.evento.so/api/public/v1/users/johndoe/profile-events',
  CURLOPT_RETURNTRANSFER => true,
  CURLOPT_HTTPHEADER => array(
    'x-evento-api-key: YOUR_API_KEY'
  ),
));

$response = curl_exec($curl);
curl_close($curl);

$result = json_decode($response, true);

if ($result['success']) {
    $hosted = 0;
    $attending = 0;
    
    foreach ($result['data'] as $event) {
        if ($event['creator']['username'] === 'johndoe') {
            $hosted++;
        } else {
            $attending++;
        }
    }
    
    echo "Hosting: $hosted events\n";
    echo "Attending: $attending events\n";
}
?>
```

## Notes

<Note>
- This endpoint returns both:
  - Events **created by** the user
  - Events the user has **RSVP'd to attend**
- You can distinguish between them by checking if the `creator.username` matches the requested username
- Events are sorted by creation date (most recent first)
- Only published, public events are returned
</Note>

<Tip>
This endpoint is ideal for building user profile pages that showcase all event activity. Use the creator information to differentiate between hosted and attending events.
</Tip>
